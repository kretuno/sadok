# VERSIONING RULES

## Єдина версія програми

У проєкті `SADOK` використовується одна спільна версія для всієї програми:

- корінь: `package.json`
- клієнт: `apps/client/package.json`
- сервер: `apps/server/package.json`

Ці три версії завжди мають бути однаковими.

## Коли піднімати версію

Використовується схема `MAJOR.MINOR.PATCH`.

- `PATCH`:
  для кожного помітного виправлення, доробки інтерфейсу, друку, логіки розрахунків, текстів, звітів, дрібних нових можливостей без зламу сумісності.
- `MINOR`:
  для нового великого модуля або суттєвого розширення можливостей програми.
- `MAJOR`:
  для змін, які ламають сумісність даних, API або вимагають окремого переходу.

## Обов'язкові дії при зміні версії

Після кожної завершеної ітерації потрібно:

1. Підняти єдину версію всієї програми.
2. Записати зміни у `CHANGELOG.txt`.
3. Оновити `SESSION_NOTES.md`.
4. Якщо потрібно, оновити `TODO.md` або іншу супровідну документацію.

## Команди

Для підняття версії використовуються команди з кореня проєкту:

- `npm run version:patch`
- `npm run version:minor`
- `npm run version:major`

Вони синхронно змінюють версію у:

- `package.json`
- `apps/client/package.json`
- `apps/server/package.json`

## Поточне практичне правило

- Після кожної помітної завершеної зміни піднімаємо `PATCH`.
- Не залишаємо зміну без запису в `CHANGELOG.txt`.
- Не допускаємо розсинхрону між коренем, клієнтом і сервером.
