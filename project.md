Ти — досвідчений fullstack-розробник. Розроби програму «Система управління дитячим садком» для Windows/macOS з такими характеристиками.

--- МОВА ---
Інтерфейс повністю українською мовою. Використовувати i18next з локаллю `uk`.

--- СТЕК ---
Frontend: React 18 + Vite + TypeScript + Tailwind CSS + shadcn/ui
Backend: Node.js (Express або Fastify) + SQLite (better-sqlite3) + Drizzle ORM
Реальний час: socket.io (чат, live-оновлення)
Друк/PDF: Puppeteer або html2canvas + jsPDF
Запуск: браузер на localhost:3000, ярлик на робочому столі (`bat`-файл або Electron shell)
Встановлення: один `exe`-файл, вибір режиму — «Сервер» або «Клієнт»

--- АРХІТЕКТУРА ---
Головний ПК:
- запускає сервер
- зберігає SQLite у `%APPDATA%/KinderAdmin/db/`
- відкриває браузер на `http://localhost:3000`

Клієнтські ПК:
- до 10 клієнтів підключаються по локальній мережі на `http://[IP]:3000`

Вимоги до ресурсів:
- RAM <= 512 MB у спокої

--- МОДУЛІ ---

1. АВТОРИЗАЦІЯ ТА РОЛІ
Ролі:
- Адміністратор
- Завгосп
- Комірник
- Медсестра
- Вихователь
- Бухгалтер
- Кухар

Функції:
- матриця дозволів: перегляд / редагування / видалення / друк по кожному модулю
- JWT-сесії
- автологін/автолог-аут через заданий період бездіяльності

2. ВНУТРІШНІЙ ЧАТ
Функції:
- особисті повідомлення
- групові чати
- лічильник непрочитаних
- звукові сповіщення
- збереження історії у БД

3. СКЛАД ПРОДУКТІВ
Таблиці:
- `products`
- `product_batches`
- `stock_movements`

Логіка:
- середня вартість (WAC) при надходженні
- списання за FIFO по `arrival_date`
- картка продукту з повною хронологією руху
- графік залишків
- інвентаризація з поясненням та записом у журнал

4. НАКЛАДНІ
Форма:
- № накладної
- дата
- постачальник (довідник)
- список позицій (продукт, кількість, ціна)

Функції:
- кнопка «Провести» оновлює `product_batches` і `stock_movements`
- перегляд і друк у форматі приходного ордера
- підтримка чернеток накладних

5. ПОСТАЧАЛЬНИКИ
Довідник:
- назва
- ЄДРПОУ
- контакти
- договори (файли)
- історія поставок

6. РЕЦЕПТИ
Функції:
- вбудована база страв
- назва
- тип страви
- вікова група
- інгредієнти: продукт + грамаж брутто/нетто
- вихід страви
- технологічна картка
- редагування грамажу окремо для груп `0-4` та `5-7`
- автоматичний розрахунок собівартості

7. МЕНЮ ДНЯ
Функції:
- дата
- кількість дітей `0-4` та `5-7`
- цільова вартість для обох груп
- поточна вартість з живим перерахунком
- рядки меню: тип прийому їжі, страва, грамаж (редагується), автовартість
- підтвердження меню → списання по FIFO, блокування
- меню на тиждень: шаблони, копіювання
- друк меню для батьків та технологічна картка для кухні

8. СКЛАД ТМЦ
Розділи:
- групові кімнати
- господарський інвентар
- спорт
- зовнішня територія
- кухня
- медкабінет
- інше

Поля:
- інвентарний номер
- назва
- розділ
- місцезнаходження
- відповідальний
- стан
- фото
- вартість

Функції:
- передача відповідальності
- інвентаризація
- списання

9. СПІВРОБІТНИКИ
Картка:
- ПІБ
- фото
- посада
- група
- контакти
- дата прийому
- роль у системі
- список закріплених ТМЦ
- документи
- кадрова історія

10. ОСОБОВІ СПРАВИ ДІТЕЙ
Функції:
- групи
- картка дитини: ПІБ, дата народження, вікова категорія, фото, батьки, зарахування
- щоденний журнал присутності
- кількість автоматично підтягується в меню

Увага:
- кількість дітей на сніданок = фактична кількість попереднього дня

11. МЕДКАБІНЕТ
Функції:
- медична картка дитини
- алергії
- хронічні захворювання
- щеплення з нагадуваннями
- огляди
- антропометрія
- аптечка садка
- строки придатності медикаментів
- журнал надання допомоги

12. ЛІЧИЛЬНИКИ
Функції:
- довідник лічильників: електро, вода, газ, тепло, інше
- введення показань
- автоматичний розрахунок споживання
- тарифи
- друк звіту за місяць / квартал / рік

13. ЗВІТИ
Усі звіти:
- переглянути
- друкувати
- зберегти PDF

Перелік:
- оборотно-сальдова відомість
- залишки
- списання
- вартість харчування
- меню
- накладні
- ТМЦ
- список дітей
- щеплення
- лічильники
- журнал дій

14. ЖУРНАЛ ДІЙ
Таблиця `audit_log`:
- user
- action_type
- entity
- old_value
- new_value
- timestamp
- ip

Функції:
- незмінний журнал
- фільтри по всіх полях
- доступний усім ролям у режимі перегляду

15. РЕЗЕРВНЕ КОПІЮВАННЯ
Функції:
- автоматично щодня у заданий час
- стислий `.sqlite.gz`
- зберігати останні `N` копій
- ручне резервне копіювання
- відновлення з файлу
- попередження на дашборді, якщо резервних копій не було понад 2 дні

16. ДАШБОРД
Картки:
- дітей сьогодні
- продуктів на складі
- меню затверджено
- ТМЦ

Попередження:
- продукти, що закінчуються
- прострочені медикаменти
- відсутність резервної копії

Графіки:
- вартість харчування за 30 днів

17. НАЛАШТУВАННЯ
Функції:
- дані садка: назва, адреса, завідувач, бухгалтер, медсестра, телефон, ЄДРПОУ, печатка

- час бекапу
- кількість копій
- автолог-аут
- назви груп дітей
