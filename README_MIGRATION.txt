ІНСТРУКЦІЯ З ПЕРЕНЕСЕННЯ ПРОЕКТУ SADOK НА WINDOWS
=================================================

Проєкт підготовлено для перенесення з macOS на Windows через SSD без локальних збірок і старих залежностей.

1. ЩО ВЖЕ ЗРОБЛЕНО
   - Видалено локальні папки `node_modules` та `dist`, щоб не переносити macOS-залежності на Windows.
   - Додано кросплатформенний запуск Electron: команда `npm run start` тепер працює і на Windows.
   - Додано команду очищення `npm run clean:portable` для майбутніх переносів між ОС.

2. ЩО ПЕРЕНОСИМО
   - Перенесіть усю папку `SADOK` на SSD або одразу відкрийте її з SSD на Windows.
   - Не видаляйте файл бази даних `apps/server/sqlite.db`, якщо хочете зберегти всі дані.

3. ЩО ВСТАНОВИТИ НА WINDOWS
   - Node.js LTS, рекомендовано 20.x або новіше.
   - Git, бажано для зручності, але не обов'язково.
   - Якщо з'явиться помилка зі збіркою нативних модулів, встановіть `Visual Studio Build Tools` з компонентом `Desktop development with C++`.

4. ПЕРШИЙ ЗАПУСК НА WINDOWS
   - Відкрийте `PowerShell` або `Windows Terminal` у папці `SADOK`.
   - Виконайте:
     `npm run install:all`
   - Дочекайтеся завершення встановлення всіх залежностей.
   - Потім запустіть проєкт:
     `npm run dev`

5. ЯКЩО ПОТРІБНО ЛИШЕ ВІДКРИТИ ELECTRON
   - Після встановлення залежностей можна запускати:
     `npm run start`

6. ВАЖЛИВІ ПРИМІТКИ
   - На Windows залежності мають встановлюватися заново, тому що `electron`, `better-sqlite3`, `esbuild` та інші пакети містять платформозалежні бінарні файли.
   - Якщо встановлення перервалося, повторіть окремо:
     `npm install`
     `cd apps/client; npm install`
     `cd ../server; npm install`
   - Якщо ви знову переноситимете проєкт між macOS і Windows, перед копіюванням виконайте:
     `npm run clean:portable`
