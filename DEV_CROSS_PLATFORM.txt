SADOK: ЩОДЕННА РОБОТА НА WINDOWS ТА macOS
=========================================

1. ГОЛОВНЕ ПРАВИЛО
   - Git переносить лише код.
   - `node_modules`, логи, локальні бази, `uploads` та тимчасові файли не переносимо між ОС.

2. ПЕРШИЙ ЗАПУСК НА НОВІЙ МАШИНІ
   - Відкрийте термінал у корені проєкту.
   - Виконайте:
     `git pull`
     `npm run install:all`
   - Потім запускайте:
     `npm run dev`

3. ЩОДЕННИЙ ЦИКЛ РОБОТИ
   - Перед початком роботи:
     `git pull`
   - Після змін:
     `git add .`
     `git commit -m "ваш коментар"`
     `git push`
   - На іншій машині перед продовженням знову:
     `git pull`

4. ЯКЩО ПІСЛЯ ПЕРЕХОДУ МІЖ ОС Є ПОМИЛКА НАТИВНОГО МОДУЛЯ
   - Для `better-sqlite3` виконайте:
     `cd apps/server`
     `npm rebuild better-sqlite3`
     `cd ../..`
   - Якщо проблема ширша, перевстановіть залежності:
     `npm run install:all`

5. ЯК ПРАЦЮВАТИ БЕЗПЕЧНІШЕ
   - Основну гілку `master` використовуйте як стабільну.
   - Для нових задач краще створювати окрему гілку:
     `git checkout -b feature/my-change`
   - Після завершення:
     `git push -u origin feature/my-change`

6. ПРИВАТНІСТЬ
   - Поточний GitHub-репозиторій має бути приватним.
   - Перед кожним першим `push` нової машини перевіряйте:
     `gh repo view kretuno/sadok --json visibility`

7. ЧОГО НЕ РОБИТИ
   - Не копіюйте `node_modules` з Windows на macOS або назад.
   - Не комітьте локальні `sqlite.db`, логи та вміст `uploads`.
   - Не працюйте одночасно на двох машинах без `git pull`, інакше легко отримати конфлікти.
